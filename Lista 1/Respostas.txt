1) Os testes desenvolvidos são manuais, pois estão sendo feitos no Postman e executados manualmente. Seriam automatizados se fossem executados automaticamente, como com o Mocha, JUnit ou o Collection Runner do Postman.


2) Sim, os testes de obter todos os posts e postar dados (GET e POST) são testes Fim-a-Fim (End-to-End), pois simulam o comportamento completo de uma interação com a API, desde a requisição até a resposta.


3) Para que os testes funcionem em modo regressão, é necessário:
	
	* Automatizar os testes para que possam ser executados sem intervenção manual.
	
	* Controlar o ambiente de testes para garantir que os testes funcionem de forma consistente.
	
	* Gerar relatórios e monitoramento dos testes, para identificar falhas e manter um histórico.
	
	* Verificar a API em ambientes de produção para garantir que alterações não impactem negativamente o funcionamento da API.
